Огляд проєкту

Мобільний додаток (iOS, Android) з функціями:

  • Авторизація — Email, Google, Apple Sign-In
  • Психологічний тест — 10 питань, аналіз через OpenAI GPT-4o-mini
  • AI-чат — текстові, голосові та фото-повідомлення
  • Підписка — RevenueCat (доступ до чату)
  • Локалізація — uk, en, es, hi, zh
  • Push-сповіщення — OneSignal

Архітектура

State Management

Використовується provider. Основні провайдери:

  • AuthProvider — користувач, вхід/вихід, синхронізація з OneSignal та RevenueCat
  • SubscriptionProvider — статус підписки, перевірка premium
  • LocaleProvider — поточна мова

Навігація

Екрани зберігаються в списку _screens в AppNavigator. Перемикання за індексом через _navigateToScreen(index). Індекси:

  • 0 — NavigationScreen (головний)
  • 1 — ChatScreen
  • 2 — PaywallScreen
  • 3 — TestIntroScreen
  • 4–13 — TestQuestionScreen (10 питань)
  • 14 — TestResultScreen
  • 15 — ShareResultScreen
  • 16 — SessionCompleteScreen

Сервіси та інтеграції

Firebase

  • Firebase Auth — email, Google, Apple
  • Cloud Firestore — chatSessions, testResults, users
  • Remote Config — OpenAI API key, системні промпти

OpenAI

API key береться з Remote Config (ключ openai_api_key). Використовуються моделі GPT-4o-mini та Whisper. Сервіс: lib/services/openai_service.dart.

RevenueCat

Підписки. Ключі в subscription_service.dart: Apple та Google API keys. Ініціалізація при першому вході користувача.

OneSignal

Push-сповіщення. App ID в onesignal_service.dart. User ID синхронізується з Firebase Auth.

Конфігурація та секрети

Важливо: Перед передачею проєкту перевірте, що секрети не закомічені або замінені на placeholder.
  • firebase_options.dart — Firebase API keys, App IDs (генерується FlutterFire CLI)
  • remote_config_service.dart — fallback OpenAI API key (краще зберігати тільки в Remote Config)
  • subscription_service.dart — RevenueCat API keys (Apple, Google)
  • onesignal_service.dart — OneSignal App ID
  • android/key.properties — keystore для підпису релізу (не в репозиторії)

Firebase

Проєкт: aipsiho. Для нового середовища: flutterfire configure.

Підпис Android

Створіть android/key.properties:

storePassword=...
keyPassword=...
keyAlias=...
storeFile=шлях/до/keystore.jks
